---
title: Raster
description: Native GPUI interfaces driven by JavaScript, TypeScript, and React.
---

Raster is a GPUI host runtime for React and TypeScript applications. It does not
embed a browser or a DOM. React commits native host mutations into Rust, Rust
retains the tree and native component owner state, and GPUI renders the result.

## What Raster Provides

- `raster/react`: a React reconciler host for GPUI-backed surfaces.
- `raster/core`: intrinsic host elements such as `View`, `Label`, `Input`, and
  `Textarea`.
- `raster/component` and `raster/components`: typed wrappers for mapped
  `gpui-component` controls.
- `unplugin-raster`: Vite, Rollup, Rolldown, and esbuild bundler integration.
- A Rust GPUI demo app with development bundle hot reload.
